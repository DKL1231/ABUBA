# 웹 애플리케이션에서 싱글톤을 사용하는 이유

- **동시성 문제 해결**: 웹 애플리케이션은 동시에 다수의 요청을 처리해야 합니다. 요청마다 객체를 새로 생성하면 메모리 낭비와 비효율이 발생합니다.
- **싱글톤 패턴 사용**: 하나의 객체로 모든 요청을 처리하여 메모리 낭비를 줄이고 성능을 향상시킬 수 있습니다.

# 싱글톤 패턴

클래스의 인스턴스가 단 하나만 생성되도록 보장하는 디자인 패턴입니다.

```java
public class SingletonService {
    private static final SingletonService instance = new SingletonService();

    public static SingletonService getInstance(){
        return instance;
    }

    private SingletonService(){
    }

    public void logic(){
        System.out.println("싱글톤 객체 로직 호출");
    }
}
생성자 접근 제어: 생성자를 private로 설정해 외부에서 새로운 객체 생성을 막습니다.
하나의 인스턴스만 존재: getInstance() 메서드를 통해서만 객체에 접근할 수 있습니다.
싱글톤 패턴의 문제점
싱글톤 패턴 구현 시 추가적인 코드가 필요합니다.
DIP 위반: 클라이언트가 구체 클래스를 의존하게 되어, DIP(Dependency Inversion Principle)를 위반할 수 있습니다.
OCP 위반 가능성: 클라이언트가 구체 클래스에 의존하므로, 변경이 어렵습니다.
테스트가 어렵고, 유연성이 떨어집니다.
싱글톤 컨테이너
스프링 컨테이너는 위의 문제를 해결하기 위해 객체 인스턴스를 하나만 생성해 관리합니다.

싱글톤 레지스트리: 싱글톤 객체를 생성하고 관리하는 기능을 제공하여, 추가적인 코드 없이 싱글톤 패턴을 구현할 수 있습니다.
빈 스코프: 기본적으로 스프링은 싱글톤 빈을 사용하지만, 요청마다 새로운 객체를 생성하는 것도 가능합니다.
싱글톤 방식의 주의사항
상태 유지를 피해야 함: 싱글톤 객체는 상태를 유지하지 않도록 설계해야 합니다.
특정 클라이언트에 의존하는 필드를 사용하면 안 됩니다.
값을 변경할 수 있는 필드를 사용하면 안 됩니다.
필드 대신 공유되지 않는 지역변수, 파라미터, ThreadLocal 등을 사용해야 합니다.
java
코드 복사
class StatefulServiceTest {

    @Test
    void statefulServiceSingleton(){
        ApplicationContext ac = new AnnotationConfigApplicationContext(TestConfig.class);
        StatefulService statefulService1 = ac.getBean("statefulService", StatefulService.class);
        StatefulService statefulService2 = ac.getBean("statefulService", StatefulService.class);

        //ThreadA: A사용자 10000원 주문
        statefulService1.order("userA", 10000);
        //ThreadB: B사용자 20000원 주문
        statefulService2.order("userB", 20000);
        //ThreadA: 사용자A 주문 금액 조회
        int price = statefulService1.getPrice();
        //ThreadA: 사용자A는 10000원을 기대했지만, 20000원이 출력
        System.out.println("price = " + price);
        Assertions.assertThat(statefulService1.getPrice()).isEqualTo(20000);
    }

    static class TestConfig{
        @Bean
        public StatefulService statefulService(){
            return new StatefulService();
        }
    }
}
StatefulService 클래스
java
코드 복사
public class StatefulService {
    private int price; //상태를 유지하는 필드

    public void order(String name, int price){
        System.out.println("name = " + name + " price = " + price);
        this.price = price;
    }

    public int getPrice(){
        return this.price;
    }
}

@Configuration과 싱글톤
스프링 컨테이너에서 싱글톤 패턴 사용 방법
@Configuration 사용: 스프링에서 @Configuration 어노테이션을 사용하면 CGLIB이라는 바이트코드 조작 라이브러리가 적용됩니다.
CGLIB의 역할: @Configuration 클래스는 CGLIB에 의해 생성된 임의의 클래스로 대체되어 싱글톤을 보장합니다.
@Bean과 싱글톤 보장: @Bean 어노테이션만 사용해도 스프링 빈으로 등록되지만, 싱글톤이 보장되지는 않습니다. 따라서 스프링 설정 정보에는 @Configuration을 사용하는 것이 좋습니다.
